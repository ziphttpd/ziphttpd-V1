はじめに

１．ZipHttpd はローカルで動かす WEB サーバです。
HTML や画像などのコンテンツを zip に圧縮したファイルを用います。
なお、自動で解凍されますので利用者は意識する必要はありません。

コンテンツを圧縮したファイルの規定の拡張子は zhd とします。
zhd は単に zip ファイルの拡張子を変更したものです。
なので、実質的に同じものである zhd, zip, jar は全て利用可能です。
以降では、より一般的である zip で記述を統一します。

また、以降では画像などの構成要素をコンテンツとし、全体をドキュメントと呼ぶこととします。

２．ZipHttpd では JavaScript で任意のテキストを保存するために WebAPI を実装しています。
ブラウザ依存の local storage は chrome で 5 MB といった最大値制限があります。
また、chrome/edge などのブラウザ間では引き継がれません。
この WebAPI では数十MBを超えるようなデータでも保存できますし、ブラウザに依存しません。

このテキストは ZipHttpd のフォルダのapi/に置かれ、USBストレージなどに置くことで持ち運ぶことが可能です。

３．ZipHttpd はGo 言語で記述しているため、多数の OS で利用可能です。
Go が利用できる OS は、Windows/Linux/iOS など多岐にわたります。 → Go のインストール 将来的には iPhone/Android などのモバイル端末での利用も検討しています。

開発リソース的に当分は Windows に注力せざるを得ません。
Windows の利用者にはビルドなどのハードルは高いという判断です。

Windows 以外では自分でビルドして使用してください。
$ git clone https://github.com/ziphttpd/ziphttpd
$ go build -o ziphttpd cmd/main.go
$ cp ziphttpd ~/.ziphttpd
