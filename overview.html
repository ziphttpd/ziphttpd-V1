概要

HTTP サーバの優れた点

HTTP サーバの優れた点についてご説明させていただきます。

ドキュメントの提供において、事実上の標準として PDF があります。
しかし PDF はあくまで印刷を目的としたフォーマットです。
印刷を目的としないドキュメントにおいては HTML が主流です。

例：技術系の文書 MDN web docs, Microsoft Docs

表現力の豊かさ

ハイパーテキストであるため、読みやすい情報提供が容易です。
HTMLゆえの自由度は他のフォーマットを寄せ付けません。

１．柔軟な情報提供が容易です

ハイパーリンクにより目的の情報へのショートカットを作れます。
情報ごとに専用のページとして分割することも可能です。
JavaScript と組み合わせて対話的な操作の実現が可能です。
ページはURLで一意であり、JavaDoc のようにヘルプとして利用可能です。

２．マルチメディアが利用可能です

動画などのマルチメディアの使用が容易です。


素材：NHKアーカイブス - CREATIVE LIBRARY - 茶畑　空撮（１）（静岡県島田市）

３．SNS や自社サイトとの連携が容易に行えます

Twitter などの SNS や自社サイトとの連携ができます。

４．将来性が期待できます

jQuery, Bootstrap, React などのフレームワークを使用できます。
今後、新たなフレームワークが出てもその導入は容易です。
HTML を使うことは、最新のUI技術に対して伸びしろを持たせることが可能です。

採用する障壁の低さ

HTML の習得は容易です。
過大なコストを必要としないことは開発費を抑制します。

１．新たな調査・開発・購入コストが不要です

自社サイトなどの開発で培われた WEB 制作技術や資産をそのまま応用できます。
そのため、新たな調査・開発・購入コストは不要です。

２．コストの見積もりが容易です

自社サイトを外注されているという場合でも、その発注先などにそのまま発注できます。
発注実績がありますので、コストの見積もりが容易です。

３．自社のサーバ能力の増強が不要です

表示させるコンテンツは利用者のマシンで稼働されます。
そのため、自社のサーバ能力を増強する必要がありません。

セキュリティの高さ

１．脆弱性の発生する余地が限られます

HTTP サーバはコンテンツを解釈・実行せず、機能が単純です。
単純であるため、脆弱性の入る余地は限られます。

２．ブラウザは選択肢が豊富です

ビュアーであるブラウザは選択肢が豊富です。
特定のブラウザに脆弱性が発見されても、他の安全なブラウザを使うことができます。

ZipHttpd の優れた点

ZipHttpd の優れた点についてご説明させていただきます。

低コスト

利用

無料です。

ドキュメント開発

標準でドキュメント開発を助けるための機能を持ちます。
テキストエディタや、Visual Studio Code を使うことで無料での開発が可能です。

ドキュメント配布

無料のリポジトリサービスが用意されています。
リポジトリはドキュメントに到達するためのサービスです。
自社の https ウェブサイトにファイル公開用のパスを作り、ホスト名を登録します。

ウェブサイトはご用意ください。
Amazon S3 を使えば無料で公開することも可能ではあります。
著作権者の信用のためには、自ドメインの取得をお勧めします。

手軽さ

インストール

ダウンロードしてきた実行ファイルを解凍するだけです。
レジストリや /etc などを使用しません。

ドキュメントの取得

ドキュメントの取得ツールで、一覧からドキュメントを選択するだけです。
取得後に ZipHttpd を再起動すると、ブラウザから参照できるようになります。
また一度選択すれば、ドキュメントが更新された時には一括でダウンロードされます。

利用

Windows では、タスクトレイで常駐する launcher が用意されています。

セキュリティの高さ

ソースが公開されています。
第三者によるレビューが可能で、安全性は二重に担保されることが期待されます。

WebAPI

ストレージ API や IndexedDB API では
・ 保存したデータをバックアップするのが困難である。
・ 異なるブラウザ、Windows/Linux など異なるプラットフォームで共有される保証はない。
・ ブラウザの都合で消去されることがある。
といった制限があり、限定的な利用に留まらざるを得ません。

ZipHttpd WebAPI により大容量データの保存が可能です。
そして、別のブラウザでもそのまま続けて利用できます。

可搬性

Go 言語で記述していますので、Windows/Linux などプラットフォームを問いません。
ポータブルなストレージにそれぞれの実行ファイルを入れることで、持ち運んでの利用も可能です。
