使用方法

準備

任意のディレクトリにファイルを置きます。

実行前のファイル構成 
D:\.ziphttpd>tree . /F
D:\.ZIPHTTPD
│  ziphttpd.exe
│  launcher.exe
│
└─docs
        prototype.zip

launchar.exe は Windows のサポートアプリ(.Net)です。
prototype.zip はドキュメント開発で使用するひな型です。
どちらも必須ではありません。

実行

ziphttpd.exe を実行します。
quit とコマンドを入力すると終了します。

実行例 
D:\.ziphttpd>ziphttpd
directory : D:\.ziphttpd
quit

D:\.ziphttpd>

実行により以下のようなファイル構成が生成されます。

実行後のファイル構成 
D:\.ziphttpd>tree . /F
D:\.ZIPHTTPD
│  ziphttpd.exe
│  launcher.exe
│  ziphttpd.json
│  portlockins.json
│
├─docs
│      prototype.zip
│      prototype.json
│
├─log
│      ziphttpd.log
│
├─static
│  └─prototype.com
│      └─prototype
├─api
│  └─prototype.com
└─system

コマンドライン

-h をつけて実行するとコマンドライン引数の説明が表示されます。

コマンドラインヘルプ 
D:\.ziphttpd> ziphttpd -h
Usage of ziphttpd:
  -config string
        configuration directory
  -docport int
        document listen port (default 58823)
  -log string
        logging directory
  -port int
        listen port (default 8823)

ディレクトリ構成

基準ディレクトリ

基本的には ZipHttpd の実行ファイルがあるディレクトリを使用します。
実行時引数で特定ディレクトリを指定できます。

まずは、ZipHttpd 設定ファイル (後述) の置き場として考えてください。

ドキュメントディレクトリ (docs)

zip ファイルをコピーするディレクトリです。

実行例で見られるように、ZipHttpd を起動すると zip ファイルに対応した json ファイルが生成されます。
これはドキュメント単位の設定ファイルです。

ログディレクトリ (log)

実行時ログ ziphttpd.log を出力するディレクトリです。
基準ディレクトリの下の log を使用しますが、実行時引数 -log で特定ディレクトリを指定できます。
/var/log 以下にログを出力させたい場合に指定します。

一般 API データディレクトリ (api)

WebAPI での登録データを格納するディレクトリです。
api/ドキュメントグループ名/ に格納します。

システム API データディレクトリ (system)

system ドキュメントグループが WebAPI で使用する予定です。
system ドキュメントグループではドキュメントに対する API も使用できるようにする想定ですので特殊です。
そのため念のために隔離しているだけで、現時点では使用されません。

静的ファイルディレクトリ (static)

ドキュメント開発作業用のフォルダです。
static/ドキュメントグループ名/ドキュメント名/以下を、zip ファイル内のファイルよりも優先的に読み出します。
ドキュメントのディレクトリの外は参照できません。
更新版をダウンロードしても静的ファイルが読み出されるため、利用時には注意してください。

ファイル構成

ZipHttpd において、設定ファイルは　json として記述します。

ZipHttpd 設定ファイル

ziphttpd.json という固定の名称のファイルです。

ziphttpd.json の例 
{
    "contenttype": {
        "html,htm": "text/html",
        "js": "text/javascript",
        "json": "text/json",
        "css": "text/css",
        "txt": "text/plain",
        "md": "text/markdown",
        "bmp": "image/bmp",
        "gif": "image/gif",
        "ico": "image/ico",
        "jpg,jpg,jpeg,jfif,pjpeg,pjp": "image/jpeg",
        "png": "image/png",
        "svg": "image/svg+xml",
        "endterminator": null
    },
    "endterminator": null
}

ドキュメントファイル

ドキュメントディレクトリに入れる zip ファイルです。

ドキュメント設定ファイル

ドキュメントファイルの拡張子を json に置き換えたファイルです。
マニフェストファイルがない場合にはpathとnameだけが設定されます。

一般的に流通している JavaDoc を表示する場合にシフトJISで書かれていたりするなどの場合があります。
その場合に、文字コードや初期表示ページを設定することでより便利に使えます。

ドキュメント設定ファイルの例 
{
    "name": "任意の名称",
    "path": "ファイル.zip",
}

キー デフォルト 機能 その他
path zip ファイル名 zip ファイル名 絶対パスで記述可能。
docgroup "common" ドキュメントグループ名 トップページに表示されます。
またURLパスの先頭に使用されます。
name zip ベース名 ドキュメント名
contentencoding "utf-8" Content-Encoding テキストの文字エンコーディングです。
contenttype なし Content-Type 拡張子別の MIME タイプを記述します。
指定しない場合、ZipHttpd 設定ファイルを参照します。
host なし 配布元のホスト名 将来的に署名や更新で使用します。→ 参照

同じドキュメントグループ名で同じドキュメント名を持つドキュメントが存在した場合、動作は不定です。

マニフェストファイル

ドキュメントファイル内に入れておく ziphttpd/config.json という固定の名称のファイルです。

docs/prototype.zip/ziphttpd/config.json 
{
    "name": "prototype",
    "docgroup": "prototype.com",
    "host": "prototype.com",
    "contentencoding": "utf-8",
    "docroot": "index.html",
    "contenttype": {
        "html,htm": "text/html",
        "js": "text/javascript",
        "css": "text/css",
        "png": "image/png",      "ico": "image/icon",
        "svg": "image/svg+xml"
    },
    "usestaticfiles": true
}

参考例に対応したドキュメントの設定ファイルは以下のように自動設定されます。

docs/prototype.json 
{
    "contentencoding": "utf-8",
    "contenttype": {
        "css": "text/css",
        "html,htm": "text/html",
        "ico": "image/icon",
        "js": "text/javascript",
        "png": "image/png",
        "svg": "image/svg+xml"
    },
    "docgroup": "prototype.com",
    "docroot": "index.html",
    "host": "prototype.com",
    "name": "prototype",
    "path": "prototype.zip",
    "usestaticfiles": true
}
